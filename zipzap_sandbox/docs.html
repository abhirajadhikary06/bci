Zipzap Docs

Detailed usage guide for fast batch image compression on local folders, local ZIP archives, and Dropbox.

Quickstart

Get from zero to your first compressed folder in under two minutes.

  1. 01 Install Zipzap via pip — requires Python 3.8 or later.
  2. 02 If you plan to use Dropbox, add your access token to a .env file in your project root.
  3. 03 Run zipzap from the command line and follow the interactive prompts, or call the Python API directly in your scripts.
  4. 04 Compressed output lands next to your source — as a folder or a _compressed.zip file.
bash 
# 1. Install
pip install zipzap

# 2. Run the interactive CLI
zipzap

# 3. Or compress a folder directly
zipzap compress ./my-images

Prerequisite

Local folder and ZIP compression work out of the box with no configuration. Dropbox support requires an access token.

Create a .env file at your project root and add your token:

.env 
DROPBOX_ACCESS_TOKEN=your_dropbox_access_token_here

Never commit your .env file. Add it to .gitignore before your first commit to avoid leaking credentials.

Installation

Zipzap is distributed on PyPI and supports Python 3.8+.

bash 
pip install zipzap

To upgrade to the latest version:

bash 
pip install --upgrade zipzap

Verify the install:

bash 
zipzap --version

CLI Reference

The zipzap command provides an interactive prompt as well as direct sub-commands for scripting and CI pipelines.

Interactive mode

Running zipzap with no arguments starts the guided prompt. It walks you through source selection and output options.

bash 
zipzap

Compress a local folder

bash 
zipzap compress ./path/to/images

Compress a local ZIP archive

bash 
zipzap compress ./path/to/images.zip

Compress a Dropbox folder

bash 
zipzap compress --dropbox /MyFolder

Flags

Flag Default Description
--quality 85 JPEG/WebP compression quality (1–100).
--format original Output format: jpeg, png, webp, avif.
--out ./out Output directory or ZIP path.
--recursive true Recurse into subdirectories.
--watch false Watch the source folder and compress on change.
--dropbox — Dropbox folder path to compress.

API Reference

The Zipzap Python API gives you full programmatic control — import and call individual functions from your scripts, notebooks, or web app.

batch_compress_local_folder

Compress all images inside a local directory. Produces a sibling _compressed folder.

python 
from zipzap.main import batch_compress_local_folder

batch_compress_local_folder(
    r"C:\path\to\images",
    quality=85,          # optional, default 85
    out_format="webp",  # optional, keeps original format by default
)

batch_compress_local_zip

Compress all images inside a ZIP archive. Output is written as <name>_compressed.zip next to the source.

python 
from zipzap.main import batch_compress_local_zip

batch_compress_local_zip(
    r"C:\path\to\images.zip",
    quality=80,
)

authenticate_dropbox / batch_compress_dropbox

Authenticate with Dropbox using the token in your .env, then compress a remote folder in-place.

python 
from zipzap.main import authenticate_dropbox, batch_compress_dropbox

dbx = authenticate_dropbox()
batch_compress_dropbox(
    dbx,
    folder_path="/Marketing/Assets",  # remote Dropbox path
    quality=85,
)

Function reference

Function Returns Description
batch_compress_local_folder None Compresses images in a local directory, writes to _compressed/.
batch_compress_local_zip None Compresses images inside a ZIP, outputs _compressed.zip.
authenticate_dropbox Dropbox Reads DROPBOX_ACCESS_TOKEN from env and returns a client.
batch_compress_dropbox None Downloads, compresses, and re-uploads images in a Dropbox folder.

Python Usage

Use Zipzap inside any Python script, Django/Flask view, or Jupyter notebook. All three compression modes are available as top-level imports.

Compress a local folder

python 
from zipzap.main import batch_compress_local_folder

batch_compress_local_folder(r"C:\path\to\images")

Compress a local ZIP

python 
from zipzap.main import batch_compress_local_zip

batch_compress_local_zip(r"C:\path\to\images.zip")

Compress a Dropbox folder

python 
from zipzap.main import authenticate_dropbox, batch_compress_dropbox

dbx = authenticate_dropbox()
batch_compress_dropbox(dbx)

Dropbox Setup

Follow these steps to connect your Dropbox account and start compressing cloud files without downloading them locally first.

  1. 01
    Go to dropbox.com/developers and create a new app. Choose Scoped access and Full Dropbox permissions.
  2. 02
    Under the Permissions tab, enable files.content.read and files.content.write.
  3. 03
    Under the Settings tab, generate an access token and copy it.
  4. 04
    Paste the token into your .env file as DROPBOX_ACCESS_TOKEN.
  5. 05
    Run zipzap compress --dropbox /YourFolder or call batch_compress_dropbox(dbx) from Python.

Token expiry: Short-lived tokens expire after 4 hours. For long-running automation, generate an offline refresh token instead via the OAuth2 flow in the Dropbox developer console.

Full Dropbox example

python 
from zipzap.main import authenticate_dropbox, batch_compress_dropbox

# Reads DROPBOX_ACCESS_TOKEN from .env automatically
dbx = authenticate_dropbox()

# Compress everything inside /Marketing/Assets on Dropbox
batch_compress_dropbox(
    dbx,
    folder_path="/Marketing/Assets",
    quality=85,
)

Notes & Tips

  • For ZIP mode, the output file is generated as <name>_compressed.zip next to the source archive.
  • Use raw strings for Windows paths to avoid backslash escaping issues: r"C:\folder\images".
  • Always keep a backup of production folders before running bulk compression — the operation modifies files in place for Dropbox mode.
  • WebP and AVIF formats yield significantly smaller files than JPEG at equivalent visual quality. Consider using --format webp for web assets.
  • Watch mode (--watch) uses filesystem events and adds near-zero CPU overhead when idle.
  • Zipzap processes images in parallel using all available CPU cores by default.
